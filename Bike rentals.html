Scaling target variables¶

To make training the network easier,each of the continuous variables are standardized i.e. shifted and scaled, such that they have zero mean and a standard deviation of 1.

The scaling factors are saved so as to go backwards when I use the network for predictions.

Time to build the network¶

Below is the network with the structure-forward pass,backwards pass, and the hyperparameters: the learning rate, the number of hidden units, and the number of training passes.

<img src="assets/neural_network.png" width=300px>

The network has two layers, a hidden layer and an output layer. The hidden layer will use the sigmoid function for activations. The output layer has only one node and is used for the regression, the output of the node is the same as the input of the node. That is, the activation function is $f(x)=x$. A function that takes the input signal and generates an output signal, but takes into account the threshold, is called an activation function.

class NeuralNetwork(object):
    def __init__(self, input_nodes, hidden_nodes, output_nodes, learning_rate):
        # Setting number of nodes in input, hidden and output layers.
        self.input_nodes = input_nodes
        self.hidden_nodes = hidden_nodes
        self.output_nodes = output_nodes

        # Initializing weights
        self.weights_input_to_hidden = np.random.normal(0.0, self.input_nodes**-0.5, 
                                       (self.input_nodes, self.hidden_nodes))

        self.weights_hidden_to_output = np.random.normal(0.0, self.hidden_nodes**-0.5, 
                                       (self.hidden_nodes, self.output_nodes))
        self.lr = learning_rate
        
        ####  Setting self.activation_function to the implemented sigmoid function ####
        #
      
        self.activation_function = lambda x : 1 / (1+ np.exp(-x))  
       
        # OR def sigmoid(x):
        #    return 0  # Replace 0 with the sigmoid calculation here
        #self.activation_function = sigmoid
                    
    
    def train(self, features, targets):
        
        n_records = features.shape[0]
        delta_weights_i_h = np.zeros(self.weights_input_to_hidden.shape)
        delta_weights_h_o = np.zeros(self.weights_hidden_to_output.shape)
        for X, y in zip(features, targets):
            #### Implemented the forward pass here ####
            ### Forward pass ###
            # Hidden layer 
            hidden_inputs = np.dot(X, self.weights_input_to_hidden) # signals into hidden layer
            hidden_outputs = self.activation_function(hidden_inputs) # signals from hidden layer

            #Output layer
            final_inputs = np.dot(hidden_outputs, self.weights_hidden_to_output) # signals into final output layer
            final_outputs = final_inputs  # signals from final output layer
            
            #### Implemented the backward pass here ####
            ### Backward pass ###

            #  Output error
            error = y-final_outputs # Output layer error is the difference between desired target and actual output.
            
            #  Calculated the hidden layer's contribution to the error
            hidden_error = np.dot(error , self.weights_hidden_to_output.T)
            
            # Backpropagated error terms 
            output_error_term = error * 1
            hidden_error_term = hidden_error * (hidden_outputs * (1-hidden_outputs))

            # Weight step (input to hidden)
            delta_weights_i_h += hidden_error_term * X[:,None]
            # Weight step (hidden to output)
            delta_weights_h_o += (output_error_term * hidden_outputs[:,None])
 
        # Updating the weights 
        self.weights_hidden_to_output += self.lr * delta_weights_h_o / n_records   # updated hidden-to-output weights with gradient descent step
        self.weights_input_to_hidden += self.lr * delta_weights_i_h / n_records    # updated input-to-hidden weights with gradient descent step
 
    def run(self, features):
        ''' Running a forward pass through the network with input features 
        
            Arguments
            ---------
            features: 1D array of feature values
        '''
        
        #### Implemented the forward pass here ####
        # Hidden layer 
        hidden_inputs = np.dot(features, self.weights_input_to_hidden)  # signals into hidden layer
        hidden_outputs = self.activation_function(hidden_inputs) # signals from hidden layer
        
        #  Output layer 
        final_inputs =np.dot(hidden_outputs, self.weights_hidden_to_output)    # signals into final output layer
        final_outputs = final_inputs             # signals from final output layer 
        
        return final_outputs

Training the network¶

Here I'll set the hyperparameters for the network. The strategy here is to find hyperparameters such that the error on the training set is low, but the n/w is not overfitting to the data. If the network is trained too long or have too many hidden nodes, it can become overly specific to the training set and will fail to generalize to the validation set. That is, the loss on the validation set will start increasing as the training set loss drops.

I'll be using Stochastic Gradient Descent (SGD) to train the network. The idea is that for each training pass, we grab a random sample of the data instead of using the whole data set. We use many more training passes than with normal gradient descent, but each pass is much faster. This ends up training the network more efficiently.

Choosing the number of iterations¶

This is the number of batches of samples from the training data used to train the network. The more iterations used, the better the model will fit the data. However, if too many iterations are used, then the model with not generalize well to other data, this is called overfitting. The idea is to find a number where the network has a low training loss, and the validation loss is at a minimum. As the network starts overfitting, it is seen that the training loss continues to decrease while the validation loss starts to increase.

Choosing the learning rate¶

This scales the size of weight updates. If this is too big, the weights tend to explode and the network fails to fit the data. A good choice to start at is 0.1. If the network has problems fitting the data, then reducing the learning rate is taken up. It has to be noted that lower the learning rate, the smaller the steps are in the weight updates and the longer it takes for the neural network to converge.

Choosing the number of hidden nodes¶

The more hidden nodes there are, the more accurate predictions the model will make. Trying a few different numbers to see how it affects the performance-hit and trial.

import sys

### Setting the hyperparameters here ###
iterations = 6000
learning_rate = 0.04
hidden_nodes = 40
output_nodes = 1

N_i = train_features.shape[1]
network = NeuralNetwork(N_i, hidden_nodes, output_nodes, learning_rate)

losses = {'train':[], 'validation':[]}
for ii in range(iterations):
    # Go through a random batch of 128 records from the training data set
    batch = np.random.choice(train_features.index, size=128)
    X, y = train_features.ix[batch].values, train_targets.ix[batch]['cnt']
                             
    network.train(X, y)
    
    # Printing out the training progress
    train_loss = MSE(network.run(train_features).T, train_targets['cnt'].values)
    val_loss = MSE(network.run(val_features).T, val_targets['cnt'].values)
    sys.stdout.write("\rProgress: {:2.1f}".format(100 * ii/float(iterations)) \
                     + "% ... Training loss: " + str(train_loss)[:5] \
                     + " ... Validation loss: " + str(val_loss)[:5])
    sys.stdout.flush()
    
    losses['train'].append(train_loss)
    losses['validation'].append(val_loss)

Thinking about your results.¶

For the three different set of parameters chosen, the difference in predicted vs actual bike riders seems to be between 22 Dec to 26 Dec probably because it's the holiday season and people have to act on the whims of their family members and not necessarily according to a pre-planned schedule. The model does well up to Dec 21st but then has trouble predicting days with during the holidays. Buyers' behaviors change during this time. However, the model does not have enough data during the holiday seasons to accurately predict during this time period. With a few extra years worth of data the model should do much better.
